# 🚀 最快上传方案

## ⚡ 方法一：直接 Git 命令（最快，推荐）

**优点**：无脚本开销，直接执行，速度最快

### 在 Cursor 终端中执行（按 `` Ctrl + ` `` 打开终端）：

```bash
# 1. 初始化（如果还没有）
git init

# 2. 添加远程仓库
git remote add origin https://github.com/changyicheng1234/-.git
# 如果已存在，用：git remote set-url origin https://github.com/changyicheng1234/-.git

# 3. 配置用户（如果还没有）
git config user.name "changyicheng1234"
git config user.email "changyicheng1234@users.noreply.github.com"

# 4. 添加文件（只添加需要的文件，排除大文件）
git add .

# 5. 提交
git commit -m "上传课程资料"

# 6. 推送
git branch -M main
git push -u origin main
```

**预计时间**：取决于文件大小和网络速度

---

## 🎯 方法二：优化后上传（更快）

**先优化 `.gitignore` 排除大文件，再上传**

### 步骤：

1. **先更新 `.gitignore` 排除大文件**（我已经帮你优化了）

2. **然后执行上传命令**（同上）

这样可以避免上传视频、压缩包等大文件，速度会快很多。

---

## 📦 方法三：分批上传（适合超大项目）

如果文件特别多，可以分批上传：

```bash
# 第一批：只上传文档和代码
git add *.md *.py *.js *.json
git commit -m "上传文档和代码文件"
git push origin main

# 第二批：上传PDF和图片
git add *.pdf *.png *.jpg
git commit -m "上传PDF和图片"
git push origin main

# 第三批：上传其他文件
git add .
git commit -m "上传其余文件"
git push origin main
```

---

## 🔐 方法四：使用 SSH（最快，但需要配置）

**如果配置了 SSH 密钥，使用 SSH 会更快**：

```bash
# 改用 SSH 地址
git remote set-url origin git@github.com:changyicheng1234/-.git

# 然后正常推送
git push -u origin main
```

**配置 SSH**：
1. 生成密钥：`ssh-keygen -t ed25519 -C "your_email@example.com"`
2. 添加到 GitHub：https://github.com/settings/keys

## ⚙️ 速度优化建议

### 1. 排除大文件（最重要！）

确保 `.gitignore` 包含：
- `*.mp4`, `*.MOV`（视频文件）
- `*.rar`, `*.zip`, `*.7z`（压缩包）
- `*.dll`（动态链接库）
- 大图片目录

### 2. 使用 Git LFS（如果必须上传大文件）

```bash
# 安装 Git LFS
git lfs install

# 跟踪大文件类型
git lfs track "*.mp4"
git lfs track "*.pdf"

# 然后正常 add 和 commit
```

### 3. 网络优化

- 使用稳定的网络连接
- 如果在中国，考虑使用代理或镜像
- 避开网络高峰期

### 4. 压缩提交

```bash
# 只提交更改的文件，而不是全部重新提交
git add -u  # 只添加已跟踪的更改
git commit -m "更新"
git push
```

## 🎯 推荐方案

**最快的方式**：**方法一（直接 Git 命令）**

1. 打开 Cursor 终端（`` Ctrl + ` ``）
2. 复制粘贴上面的命令
3. 按 Enter 执行

**如果文件很多很大**：**方法二（优化后上传）**

1. 先确保 `.gitignore` 排除了大文件
2. 然后执行方法一的命令

---

## ⚠️ 注意事项

1. **首次上传可能较慢**：文件多的话需要时间
2. **大文件限制**：GitHub 单个文件限制 100MB
3. **认证问题**：可能需要输入 Personal Access Token
4. **网络中断**：如果中断，可以重新执行 `git push`

## 🆘 如果上传失败

### 问题1：认证失败
```bash
# 使用 Personal Access Token
# 用户名：changyicheng1234
# 密码：你的 GitHub Token
```

### 问题2：文件太大
```bash
# 检查大文件
git ls-files | xargs ls -lh | sort -k5 -hr | head -20

# 从 Git 中移除大文件
git rm --cached 大文件路径
git commit -m "移除大文件"
```

### 问题3：网络超时
```bash
# 增加缓冲区大小
git config http.postBuffer 524288000

# 或者使用 SSH
git remote set-url origin git@github.com:changyicheng1234/-.git
```
