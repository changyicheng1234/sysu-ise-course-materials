## ⚠️ 重要注意事项

### 文件大小限制

**⚠️ 严禁提交超过100MB的文件！**

- GitHub 不支持上传单个超过 100MB 的文件
- 本仓库未启用 git-lfs（Git Large File Storage）
- 如果提交了超过 100MB 的文件，commit 将无效，可能导致推送失败

### 如何处理大文件

如果发现超过 100MB 的文件，请采取以下措施：

1. **压缩文件**：将大文件压缩后再上传
2. **分割文件**：将大文件分割成多个小文件
3. **使用外部存储**：将大文件上传到网盘，在 README 中添加下载链接
4. **删除文件**：如果是临时文件或不必要的文件，直接删除

## 🔍 检查大文件工具使用

### 工具说明

`检查大文件.py` 用于扫描项目目录，查找超过 100MB 的文件，帮助您在提交前发现潜在问题。

### 使用方法

1. **直接运行脚本**：
   ```bash
   python 检查大文件.py
   ```

2. **脚本会自动**：
   - 扫描整个项目目录（自动跳过 `.git` 目录）
   - 查找所有超过 100MB 的文件
   - 显示文件列表和统计信息

3. **输出示例**：
   ```
   ================================================================================
   检查大文件工具
   ================================================================================
   正在扫描目录: D:\学习\中山大学智能工程学院本科生课程作业
   查找超过 100MB 的文件...
   --------------------------------------------------------------------------------
   
   扫描完成！
   总文件数: 1234
   总大小: 2.5 GB
   超过 100MB 的文件数: 3
   ================================================================================
   
   发现 3 个超过 100MB 的文件：
   
   1.   150.23 MB - 课程A/视频/实验演示.mp4
   2.   120.45 MB - 课程B/资料/完整课件.pdf
   3.   105.67 MB - 课程C/代码/大型数据集.zip
   ```

4. **处理结果**：
   - 如果发现大文件，请按照上述"如何处理大文件"的方法处理
   - 如果没有发现大文件，可以安全地提交代码

### 注意事项

- 脚本会自动跳过 `.git` 目录，不会扫描 Git 历史记录
- 如果遇到权限错误，脚本会跳过无法访问的文件
- 脚本会显示文件的总大小和占比，帮助您了解项目规模

## 📝 更新README资料工具使用

### 工具说明

`更新README资料.py` 用于自动更新各课程目录的 README 文件中的"已有资料"部分，并同步更新 `收录内容.md` 文件。

### 主要功能

1. **自动扫描课程目录**：识别所有包含 `README.md` 的课程目录
2. **智能分类文件**：根据文件名和路径自动分类（作业、实验、期末、课件等）
3. **更新已有资料部分**：只更新"已有资料"部分，保留其他所有内容（选课建议、考试攻略等）
4. **同步更新收录内容**：自动更新 `收录内容.md` 文件，保持目录结构一致

### 使用方法

1. **直接运行脚本**：
   ```bash
   python 更新README资料.py
   ```

2. **脚本会自动**：
   - 扫描所有课程目录
   - 分析每个目录中的文件
   - 更新各课程的 README.md 中的"已有资料"部分
   - 更新 `收录内容.md` 文件

3. **输出示例**：
   ```
   开始更新课程README文件...
   
   找到 25 个课程目录
   
   ✅ C++程序设计: README已更新（仅更新已有资料部分）
   ✅ 数据结构与算法: README已更新（仅更新已有资料部分）
   ✅ 离散数学: README已更新（仅更新已有资料部分）
   ...
   
   ✅ 收录内容.md 已更新
   
   完成！共处理 25 个课程目录
   ```

### 重要说明

- ✅ **只更新"已有资料"部分**：脚本会保留 README 中的其他所有内容，包括：
  - 选课建议
  - 考试攻略
  - 课程介绍
  - 其他手动编辑的内容

- ✅ **自动清理文件名**：脚本会自动移除学号等冗余信息，使文件名更简洁

- ✅ **智能去重**：自动去除重复的文件描述

- ⚠️ **特殊目录处理**：`00-大一上期末` 和 `01-大一下期末` 目录不需要 README.md，但会在 `收录内容.md` 中单独列出

## 🔄 日常维护流程

### 关于权限和贡献流程

**重要说明**：
- ✅ **不需要设置特殊权限**：任何人都可以 Fork 本项目，这是 GitHub 的默认行为
- ✅ **别人不能直接修改主仓库**：贡献者只能在自己的 Fork 副本中上传和修改文件
- ✅ **需要管理者审查**：贡献者通过 Pull Request（PR）提交后，**必须由管理者审查并合并**，才能添加到主仓库
- ✅ **安全可控**：所有更改都需要经过您的审查，确保项目质量和规范

**贡献流程**：
1. 贡献者 Fork 项目 → 在自己的副本中上传文件 → 提交 Pull Request
2. 管理者收到 PR 通知 → 审查 PR → 检查文件 → 合并或提出修改意见

### 审查 Pull Request 的流程

当收到新的 Pull Request 时，请按以下步骤审查：

1. **查看 PR 内容**：
   - 在 GitHub 上打开 PR 页面
   - 查看文件变更（Files changed）
   - 了解贡献者添加了哪些文件

2. **检查大文件**：
   - 在 PR 的"Files changed"页面中，GitHub 会显示文件大小
   - 如果发现有超过 100MB 的文件，**不要合并**，在 PR 中留言要求贡献者处理
   - 也可以下载 PR 分支到本地，运行 `检查大文件.py` 进行检查

3. **检查文件结构**：
   - 确认文件放在正确的课程目录下
   - 确认文件命名规范（建议使用有意义的文件名）
   - 确认没有添加不相关或敏感文件

4. **合并 PR**：
   - 如果一切正常，点击"Merge pull request"合并
   - 建议在合并前添加评论，感谢贡献者

5. **合并后更新**：
   - 合并 PR 后，拉取最新代码到本地
   - 运行 `更新README资料.py` 更新资料列表
   - 提交更新后的 README 和收录内容

### 接收新贡献时的检查流程（本地）

合并 PR 后，在本地进行以下检查：

1. **拉取最新代码**：
   ```bash
   git pull origin main
   ```

2. **检查大文件**：
   ```bash
   python 检查大文件.py
   ```
   - 如果发现超过 100MB 的文件，联系贡献者处理或删除

3. **检查文件结构**：
   - 确认文件放在正确的课程目录下
   - 确认文件命名规范（建议使用有意义的文件名）

4. **更新README**：
   ```bash
   python 更新README资料.py
   ```
   - 运行脚本自动更新资料列表

5. **检查更新结果**：
   - 查看各课程的 README.md，确认"已有资料"部分已更新
   - 查看 `收录内容.md`，确认目录结构正确

6. **提交更改**：
   ```bash
   git add .
   git commit -m "更新资料列表：添加了XX课程的XX资料"
   git push origin main
   ```

### 定期维护建议

- **每月**：运行一次 `更新README资料.py`，保持资料列表最新
- **每月**：运行一次 `检查大文件.py`，检查是否有大文件
- **每次提交前**：运行 `检查大文件.py`，确保没有超过 100MB 的文件
