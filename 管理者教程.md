## ⚠️ 重要注意事项

### 文件大小限制

**⚠️ 严禁提交超过100MB的文件！**

- GitHub 不支持上传单个超过 100MB 的文件
- 本仓库未启用 git-lfs（Git Large File Storage）
- 如果提交了超过 100MB 的文件，commit 将无效，可能导致推送失败

### 如何处理大文件

如果发现超过 100MB 的文件，请采取以下措施：

1. **压缩文件**：将大文件压缩后再上传
2. **分割文件**：将大文件分割成多个小文件
3. **使用外部存储**：将大文件上传到网盘，在 README 中添加下载链接
4. **删除文件**：如果是临时文件或不必要的文件，直接删除

## 🔍 检查大文件工具使用

### 工具说明

`检查大文件.py` 用于扫描项目目录，查找超过 100MB 的文件，帮助您在提交前发现潜在问题。

### 使用方法

1. **直接运行脚本**：
   ```bash
   python 检查大文件.py
   ```

2. **脚本会自动**：
   - 扫描整个项目目录（自动跳过 `.git` 目录）
   - 查找所有超过 100MB 的文件
   - 显示文件列表和统计信息

3. **输出示例**：
   ```
   ================================================================================
   检查大文件工具
   ================================================================================
   正在扫描目录: D:\学习\中山大学智能工程学院本科生课程作业
   查找超过 100MB 的文件...
   --------------------------------------------------------------------------------
   
   扫描完成！
   总文件数: 1234
   总大小: 2.5 GB
   超过 100MB 的文件数: 3
   ================================================================================
   
   发现 3 个超过 100MB 的文件：
   
   1.   150.23 MB - 课程A/视频/实验演示.mp4
   2.   120.45 MB - 课程B/资料/完整课件.pdf
   3.   105.67 MB - 课程C/代码/大型数据集.zip
   ```

4. **处理结果**：
   - 如果发现大文件，请按照上述"如何处理大文件"的方法处理
   - 如果没有发现大文件，可以安全地提交代码

### 注意事项

- 脚本会自动跳过 `.git` 目录，不会扫描 Git 历史记录
- 如果遇到权限错误，脚本会跳过无法访问的文件
- 脚本会显示文件的总大小和占比，帮助您了解项目规模

## 📝 更新README资料工具使用

### 工具说明

`更新README资料.py` 用于自动更新各课程目录的 README 文件中的"已有资料"部分，并同步更新 `收录内容.md` 文件。

### 主要功能

1. **自动扫描课程目录**：识别所有包含 `README.md` 的课程目录
2. **智能分类文件**：根据文件名和路径自动分类（作业、实验、期末、课件等）
3. **更新已有资料部分**：只更新"已有资料"部分，保留其他所有内容（选课建议、考试攻略等）
4. **同步更新收录内容**：自动更新 `收录内容.md` 文件，保持目录结构一致

### 使用方法

1. **直接运行脚本**：
   ```bash
   python 更新README资料.py
   ```

2. **脚本会自动**：
   - 扫描所有课程目录
   - 分析每个目录中的文件
   - 更新各课程的 README.md 中的"已有资料"部分
   - 更新 `收录内容.md` 文件

3. **输出示例**：
   ```
   开始更新课程README文件...
   
   找到 25 个课程目录
   
   ✅ C++程序设计: README已更新（仅更新已有资料部分）
   ✅ 数据结构与算法: README已更新（仅更新已有资料部分）
   ✅ 离散数学: README已更新（仅更新已有资料部分）
   ...
   
   ✅ 收录内容.md 已更新
   
   完成！共处理 25 个课程目录
   ```

### 重要说明

- ✅ **只更新"已有资料"部分**：脚本会保留 README 中的其他所有内容，包括：
  - 选课建议
  - 考试攻略
  - 课程介绍
  - 其他手动编辑的内容

- ✅ **自动清理文件名**：脚本会自动移除学号等冗余信息，使文件名更简洁

- ✅ **智能去重**：自动去除重复的文件描述

- ⚠️ **特殊目录处理**：`大一上期末` 和 `大一下期末` 目录不需要 README.md，但会在 `收录内容.md` 中单独列出

## 🔄 日常维护流程

### 接收新贡献时的检查流程

1. **检查大文件**：
   ```bash
   python 检查大文件.py
   ```
   - 如果发现超过 100MB 的文件，联系贡献者处理

2. **检查文件结构**：
   - 确认文件放在正确的课程目录下
   - 确认文件命名规范（建议使用有意义的文件名）

3. **更新README**：
   ```bash
   python 更新README资料.py
   ```
   - 运行脚本自动更新资料列表

4. **检查更新结果**：
   - 查看各课程的 README.md，确认"已有资料"部分已更新
   - 查看 `收录内容.md`，确认目录结构正确

5. **提交更改**：
   - 如果一切正常，可以提交更改
   - 建议在提交信息中说明更新的内容

### 定期维护建议

- **每月**：运行一次 `更新README资料.py`，保持资料列表最新
- **每月**：运行一次 `检查大文件.py`，检查是否有大文件
- **每次提交前**：运行 `检查大文件.py`，确保没有超过 100MB 的文件
