# README自动更新说明

## 功能说明

为了方便大家补充资料并保持README内容整洁，我们创建了一个自动化脚本 `更新README资料.py`。

### 主要特点

1. **自动扫描目录**：脚本会自动扫描每个课程目录下的所有文件
2. **智能分类**：根据文件名和路径自动识别文件类型（作业、实验、报告、课件等）
3. **自动生成描述**：根据文件命名自动生成简洁的描述
4. **无需手动记录**：不需要手动记录谁添加了什么资料，脚本会自动更新

## 使用方法

### 方法一：双击批处理文件（推荐）

直接双击 `运行更新README.bat` 文件，脚本会自动运行并更新所有课程的README。

### 方法二：命令行运行

```bash
python 更新README资料.py
```

## 工作原理

1. **文件分类**：脚本会根据文件名中的关键词自动分类：
   - 包含"作业"、"homework"等 → **作业**
   - 包含"实验"、"exp"、"lab"等 → **实验**
   - 包含"期末"、"final"等 → **期末**
   - 包含"报告"、"report"等 → **报告**
   - 包含"课件"、"ppt"等 → **课件**
   - 包含"教材"、"textbook"等 → **教材**
   - 等等...

2. **文件描述生成**：
   - 自动清理文件名中的学号、姓名等冗余信息
   - 对于子目录中的文件，显示目录名
   - 对于单个文件，显示清理后的文件名

3. **README更新**：
   - 保留"选课建议"部分不变
   - 自动更新"资料介绍"部分
   - 移除旧的"资料扩展记录"部分

## 文件命名建议

为了让脚本更好地识别和分类文件，建议使用以下命名方式：

- **作业**：`作业1.pdf`、`homework2.docx`、`第三次作业.pdf`
- **实验**：`实验1/`、`exp2/`、`lab3/`
- **期末**：`期末报告.pdf`、`final_exam/`
- **报告**：`课程报告.docx`、`report.pdf`
- **课件**：`第1章.ppt`、`lecture_slides.pptx`

## 注意事项

1. 脚本会忽略以下文件/目录：
   - `README.md`
   - `.git`、`__pycache__` 等系统目录
   - 以 `~$` 开头的临时文件

2. 脚本会限制递归深度（最多3层），避免扫描过深的目录结构

3. 每个类别最多显示15项，超过的会显示"还有X项"的提示

## 示例

运行脚本前：
```markdown
### 已有资料

- **作业资料**：包含第二次、第三次、第四次作业的文档和PDF
```

运行脚本后：
```markdown
### 已有资料

- **作业**：
  - 第二次作业
  - 第三次作业
  - 第四次作业
```

## 常见问题

**Q: 如果文件分类不正确怎么办？**  
A: 可以在脚本的 `FILE_CATEGORIES` 字典中添加更多关键词，或者修改文件名使其包含相应的关键词。

**Q: 可以只更新某个课程的README吗？**  
A: 可以修改脚本中的 `main()` 函数，指定要更新的课程目录。

**Q: 脚本会删除我的文件吗？**  
A: 不会。脚本只读取和更新README.md文件，不会删除或修改任何其他文件。
