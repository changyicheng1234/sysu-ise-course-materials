# 贡献指南

欢迎为这个课程资料共享项目贡献内容！

## 📝 如何贡献

### 1. 添加课程资料

**最简单的方式**：

1. **Fork 本项目**
2. **添加资料**：将你的课程资料文件/文件夹放入对应的课程目录
3. **提交更改**：提交你的更改
4. **自动更新**：GitHub Actions 会自动更新 README，无需手动编辑

### 2. 文件命名建议

为了让脚本更好地识别和分类文件，建议使用以下命名方式：

| 文件类型 | 推荐命名 | 示例 |
|---------|---------|------|
| 作业 | 包含"作业"、"homework"、"hw" | `作业1.pdf`、`homework2.docx` |
| 实验 | 包含"实验"、"exp"、"lab" | `实验1/`、`exp2/`、`lab3/` |
| 期末 | 包含"期末"、"final"、"exam" | `期末报告.pdf`、`final_exam/` |
| 报告 | 包含"报告"、"report" | `课程报告.docx`、`report.pdf` |
| 课件 | 包含"课件"、"ppt"、"slides" | `第1章.ppt`、`lecture_slides.pptx` |
| 教材 | 包含"教材"、"textbook"、"book" | `数字图像处理教材.pdf` |

### 3. 添加新课程

如果你想添加一个新的课程目录：

1. 创建课程目录（如 `新课程名称/`）
2. 在目录中创建 `README.md` 文件，格式如下：

```markdown
# 课程名称

## 选课建议

> 待补充：请在此处添加关于该课程的选课建议...

## 资料介绍

> 💡 **提示**：本目录的资料列表由脚本自动生成。添加新资料后，GitHub Actions 会自动更新此部分。

### 已有资料

*暂无资料*
```

3. 将课程资料放入该目录
4. 提交 Pull Request

### 4. 更新选课建议

如果你想更新某个课程的选课建议：

1. 编辑对应课程目录下的 `README.md`
2. 修改"选课建议"部分（脚本不会修改这部分）
3. 提交更改

## 🤖 自动化说明

### 更新 README

添加资料后，需要运行脚本更新 README：

1. **运行更新脚本**：双击 `运行更新README.bat` 或运行 `python 更新README资料.py`
2. **检查更新**：查看 README 中的"已有资料"部分是否已更新
3. **编辑选课建议**（可选）：手动编辑 README 的"选课建议"部分
4. **提交更改**：提交到 GitHub

**注意**：脚本只会更新"已有资料"部分，"选课建议"部分需要手动编辑。

### 本地测试

如果你想在本地测试脚本：

```bash
# 方法1：使用批处理文件（Windows）
运行更新README.bat

# 方法2：直接运行Python脚本
python 更新README资料.py
```

## 📋 提交规范

### Commit 信息

建议使用以下格式：

```
feat: 添加[课程名称]的[资料类型]
docs: 更新[课程名称]的选课建议
fix: 修复[问题描述]
```

示例：
- `feat: 添加分布式计算的实验3代码`
- `docs: 更新图像处理的选课建议`
- `fix: 修复README格式问题`

### Pull Request

提交 PR 时，请：

1. **描述清楚**：说明你添加了什么资料，属于哪个课程
2. **检查格式**：确保文件命名符合规范
3. **等待检查**：GitHub Actions 会自动检查并更新 README

## ⚠️ 注意事项

1. **隐私保护**：
   - 建议移除文件名中的学号、姓名等个人信息
   - 可以使用脚本 `移除学号重命名.py` 批量处理

2. **文件大小**：
   - 大文件（>100MB）建议使用 Git LFS 或外部链接
   - 避免提交不必要的临时文件

3. **版权问题**：
   - 确保你有权分享这些资料
   - 尊重原作者的版权

4. **文件格式**：
   - 优先使用开放格式（PDF、Markdown等）
   - 避免使用专有格式（如 .docx，如果可能的话）

## 🐛 问题反馈

如果遇到问题：

1. 查看 [Issues](https://github.com/your-repo/issues) 是否已有相关问题
2. 如果没有，创建新的 Issue 描述问题
3. 如果是脚本问题，请提供错误信息和相关文件
