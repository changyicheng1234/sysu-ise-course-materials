# 如何运行上传脚本

## 方法一：双击运行（最简单）

### 步骤：

1. **打开文件资源管理器**
   - 按 `Win + E` 打开文件资源管理器
   - 或者直接打开"此电脑"

2. **找到项目文件夹**
   - 导航到：`D:\学习\中山大学智能工程学院本科生课程作业`

3. **找到批处理文件**
   - 在文件夹中找到 `上传到GitHub.bat` 文件
   - 它应该显示为一个齿轮图标或者批处理文件图标

4. **双击运行**
   - 直接用鼠标左键双击 `上传到GitHub.bat` 文件
   - 会弹出一个黑色的命令行窗口
   - 脚本会自动执行所有步骤

5. **等待完成**
   - 脚本会自动执行：初始化仓库、添加文件、提交、推送等步骤
   - 如果提示输入用户名和密码，按照提示操作即可

### 如果双击没有反应：

- **右键点击文件** → 选择"以管理员身份运行"
- 或者右键点击 → 选择"编辑"，查看文件内容是否正确

## 方法二：在 Cursor 中运行

### 在 Cursor 的终端中运行：

1. 按 `` Ctrl + ` `` 打开终端（或者点击菜单：终端 → 新建终端）

2. 在终端中输入：

```bash
.\上传到GitHub.bat
```

然后按 Enter 键执行。

---

## 方法三：使用 PowerShell 运行

1. 按 `Win + X`，选择"Windows PowerShell"或"终端"

2. 切换到项目目录：

```powershell
cd "D:\学习\中山大学智能工程学院本科生课程作业"
```

3. 运行脚本：

```powershell
.\上传到GitHub.bat
```

或者运行 PowerShell 脚本：

```powershell
.\上传到GitHub.ps1
```

（如果提示执行策略限制，先运行：`Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`）

---

## 方法四：手动执行命令（最可靠）

如果脚本运行有问题，可以手动在终端中执行以下命令：

```bash
# 1. 确保在正确的目录
cd "D:\学习\中山大学智能工程学院本科生课程作业"

# 2. 初始化 Git 仓库（如果还没有）
git init

# 3. 添加远程仓库
git remote add origin https://github.com/changyicheng1234/-.git
# 如果已经添加过，使用：
# git remote set-url origin https://github.com/changyicheng1234/-.git

# 4. 配置用户信息（如果还没有）
git config user.name "changyicheng1234"
git config user.email "changyicheng1234@users.noreply.github.com"

# 5. 添加所有文件
git add .

# 6. 提交
git commit -m "更新：上传课程资料到 GitHub"

# 7. 推送到 GitHub
git branch -M main
git push -u origin main
```

## 常见问题

### Q: 双击后窗口一闪就关闭了？
A: 可能是脚本执行出错。右键点击文件 → "编辑"，查看错误信息。或者使用"方法四"手动执行命令。

### Q: 提示"无法执行脚本"？
A: 可能是 PowerShell 执行策略限制。在 PowerShell 中运行：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q: 提示需要输入用户名和密码？
A: 
- 用户名：`changyicheng1234`
- 密码：使用 GitHub Personal Access Token（不是 GitHub 登录密码）
- 获取 Token：访问 https://github.com/settings/tokens

### Q: 文件太大无法推送？
A: GitHub 限制单个文件不能超过 100MB。检查 `.gitignore` 文件，确保大文件被忽略。
