# 更新说明：本地维护模式

## 📝 更改内容

根据你的要求，已将所有更新改为**本地维护模式**：

### ✅ 已完成的更改

1. **移除 GitHub Actions 自动更新**
   - 删除了 `.github/workflows/update-readme.yml`
   - 不再使用 GitHub Actions 自动更新

2. **修改脚本逻辑**
   - 脚本现在**只更新"已有资料"部分**
   - **完全保留"选课建议"部分**，脚本不会修改
   - 更新了脚本注释和提示信息

3. **更新文档**
   - 创建了 `使用说明.md` - 详细的使用指南
   - 更新了 `README.md` - 移除 GitHub Actions 相关说明
   - 更新了 `CONTRIBUTING.md` - 更新贡献流程

## 🚀 使用方式

### 工作流程

1. **添加资料** → 将文件放入对应课程目录
2. **运行脚本** → 双击 `运行更新README.bat` 或运行 `python 更新README资料.py`
3. **检查更新** → 查看 README 的"已有资料"部分是否已更新
4. **编辑选课建议**（可选）→ 手动编辑 README 的"选课建议"部分
5. **提交到 GitHub** → 
   ```bash
   git add .
   git commit -m "更新课程资料"
   git push
   ```

## 🔍 脚本行为

### ✅ 会更新

- **"已有资料"部分**：根据目录中的文件自动生成和更新

### ❌ 不会更新

- **"选课建议"部分**：完全保留，需要手动编辑
- **其他部分**：README 的其他内容保持不变

## 📋 README 结构

每个课程的 README.md 结构：

```markdown
# 课程名称

## 选课建议

> （这部分只能手动编辑，脚本不会修改）

## 资料介绍

> 💡 **提示**：本目录的资料列表由脚本自动生成...

### 已有资料

- **作业**：...
- **实验**：...
（这部分由脚本自动更新）
```

## ⚠️ 重要提示

1. **选课建议不会被修改**：
   - 脚本只会更新"已有资料"部分
   - "选课建议"部分需要手动编辑
   - 脚本运行后，选课建议的内容完全保留

2. **本地运行**：
   - 所有更新都在本地完成
   - 需要手动提交到 GitHub
   - 不再有自动提交功能

3. **定期更新**：
   - 建议每次添加资料后运行一次脚本
   - 然后再手动编辑选课建议（如需要）
